## 软件工程

### GitHub 修复 git push 关键远程代码执行漏洞

**事件：** GitHub 披露并修复了一个严重的远程代码执行漏洞（CVE‑2026‑3854）。该漏洞允许拥有推送权限的攻击者在 git push 过程中注入恶意命令，绕过沙箱执行服务器端代码。漏洞在报告后两小时内被修复。

**解读：** 该漏洞暴露了现代软件开发核心流程（CI/CD）中的安全脆弱性。它提升了企业对供应链安全的风险评估等级，可能会推动对代码提交和自动化流程增加更严格的审核与安全扫描，从而改变开发运维的成本结构。

评分 87 · 来源 [The GitHub Blog - Alexis Wales](https://github.blog/security/securing-the-git-push-pipeline-responding-to-a-critical-remote-code-execution-vulnerability/)

---

### Martin Fowler 提出结构化提示驱动开发 (SPDD)

**事件：** 软件工程思想领袖 Martin Fowler 撰文介绍了一种名为“结构化提示驱动开发”（SPDD）的新方法。该方法旨在通过结构化的提示词，使大型语言模型辅助的软件开发过程更具可治理性、可重复性和可预测性。

**解读：** SPDD 的提出，试图将 LLM 从个人生产力工具提升为团队协作和工程管理的核心环节。这可能改变软件开发的工程效率和质量控制标准，为评估 AI 在软件工程中的应用成熟度提供了新的框架，可能催生新的开发者工具。

评分 84 · 来源 [Martin Fowler](https://martinfowler.com/articles/structured-prompt-driven/)
