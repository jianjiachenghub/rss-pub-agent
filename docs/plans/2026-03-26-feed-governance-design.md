# 个人日报抓取治理设计

## 目标

把当前偏向“大而全信息流”的抓取入口，改成更适合个人日报的输入层：

- 主日报候选池以高信噪比源为主
- 高频热榜和大杂烩源默认降级为观察池
- 单一来源不能再主导全局候选池
- 每个分类在抓取层就获得基础覆盖

## 方案

### 1. 源分层

为每个 feed 增加 3 个治理字段：

- `tier`: `core` / `signal` / `watch`
- `dailyCap`: 单源 24h 最大入池条数
- `keepInMainPool`: 是否进入主日报候选池

其中：

- `core` 用于官方博客、权威媒体、关键技术源
- `signal` 用于社区、行业媒体、补充信号源
- `watch` 用于热榜、社交源、聚合大流量源

### 2. 抓取策略

- 主池只抓 `keepInMainPool !== false` 的源
- 不再使用“`folo-list >= 50` 就跳过其他源”的熔断逻辑
- 主池结果不足时，才从 `watch` 源回填
- 所有源统一执行：
  - 24 小时过滤
  - 单源限额
  - 全局去重
  - 按分类上限再次截断

### 3. 配置策略

- `folo-list` 保留，但默认退出主日报主池，作为观察池后备
- 高频热榜类源默认设为 `watch`
- 权威低频源设为 `core`
- 社区和行业媒体设为 `signal`

## 预期效果

- 原始候选集从“可能被单一聚合源淹没”收敛到“结构化、可控的日报输入”
- 日报更接近“个人多分类精选”，而不是“AI 大杂烩”
- 后续可以在这个基础上继续做：
  - `secondaryItems` 重分类
  - Prompt 兴趣去 AI 偏置
  - 投资/财经单独洞察增强
