# 精选后补全正文与事件/解读结构设计

日期：2026-03-28

## 背景

当前日报主解读区存在两个根本问题：

1. 很多条目只有抓取摘要，没有拿到正文或原站图片，导致“新闻本身”信息量不足。
2. 在内容偏薄时仍然强行生成四格分析，结果是“分析”比事实更长，且容易出现英文事实、空泛判断或误导性解读。

本次改造只调整内容生成链，不改 `daily.md` 的整体结构，也不改网页渲染方式。

## 目标

- 在精选后只对候选条目补全正文与原站图片
- 主深度解读区从四格改为 `事件 + 解读`
- 只有正文够厚、信息明确、模型把握足够高时才写 `解读`
- 论坛提问、内容过薄、模型拿不准的条目不再进入主深度解读区
- `更多 24h 资讯` 区块保持原有结构不变
- 主日报条数从固定 `18` 改为 `基准值 + 动态扩容`

## 页面与输出

### 保持不变

- `daily.md` 的整体结构
- 网页对 `daily.md` 的渲染方式
- `更多 24h 资讯` 的位置与列表样式

### 改动

主深度解读区单条内容改为：

- `事件：`
- `解读：`

`解读` 不再拆分成 `影响 / 判断 / 动作`，而是合并为一段更紧凑的判断更新。

## 新的内容标准

### 事件

- 必须是中文
- 优先基于补全后的正文写，不准只把标题换个说法
- 不允许英文事实直接落盘
- 长度控制在短而实，不展开宏大评论

### 解读

- 只在满足以下条件时才生成：
  - 正文补全后仍有足够信息密度
  - 条目不是论坛求助、问答、闲聊、Show HN/Ask HN 一类内容
  - 模型可以明确提炼“这条信息为什么会改变判断”
- 将影响、判断、后续信号合并为一段
- 若模型无法稳定判断，则留空，不强写

### 不进入主深度解读区的条目

- V2EX 等论坛提问、求助、讨论串
- 只有一句摘要的条目
- 正文补全失败且信息仍然很薄的条目
- 模型信心不足、容易误导的条目

这些条目保留在 `更多 24h 资讯`。

## 抓取链改造

### 第一段：候选抓取

保留现有快速抓取，用于：

- 去重
- 预筛
- 初步打分

### 第二段：精选后补全

在 `score` 之后新增 `enrichSelected` 节点，仅对候选池做补全：

- 抓取原站 HTML
- 提取正文段落
- 提取 `og:image` / `twitter:image` / 正文主图
- 过滤掉纯装饰图、logo、人物照等低信息量图片

补全结果回写到候选条目中，供后续 `insight` 使用。

## 图片策略

仅使用原站自带图片，不额外补图。

图片通过信息量筛选后才进入主解读区。优先保留：

- 模型榜单图
- benchmark / chart / graph
- 财务图表
- 路线图、对比图、地图、图解

默认丢弃：

- logo
- 人物照
- 宣传图
- 装饰性头图

## 条数策略

不再固定为 `18` 条，而是采用 `基准值 + 动态扩容`：

- 基准值使用当前 `topN`
- 正常日扩到 `20`
- 事件密集日扩到 `24`
- 极端日上限 `30`

扩容依据不是“新闻数量多”，而是：

- 高分条目是否足够多
- 补全文后是否仍然有足够多条目值得写 `解读`

为避免前一轮排序误差，`score` 先选出一个更大的候选池给 `enrichSelected` 补全，再由 `insight` 决定最终进入主深度解读区的条数。

## 节点改造

### score

- 不再只选固定 `topN`
- 先选一个较大的高分候选池，供后续补全

### enrichSelected

- 新增节点
- 输入：`score` 选出的候选池
- 输出：补全后的候选条目

### insight

- 输入：补全后的候选池
- 输出：真正进入主解读区的 `insights`
- 对不满足条件的条目不生成 `解读`

### generateDaily

- 仅改单条渲染内容
- 从四格切换成 `事件 + 解读`
- `更多 24h 资讯` 原样保留

## 验证

需要至少覆盖以下验证：

1. 单元测试
   - 内容薄条目会被识别为不适合深度解读
   - `事件` 会优先落成中文
   - 信息量不足的图片不会被带入主解读区

2. 类型与脚本验证
   - `npm test`
   - `npx tsc --noEmit -p tsconfig.json`

3. 实跑验证
   - 使用 `--resume-from-raw` 重跑一条日报
   - 确认论坛提问等薄内容不再进入主深度解读区
   - 确认主解读区渲染为 `事件 + 解读`
